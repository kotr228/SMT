# 🧪 Тестування NetworkCore на одному комп'ютері

## Мета
Протестувати роботу мережевого ядра без наявності другого фізичного пристрою.

## Принцип
Запускаємо два екземпляри NetworkCore на одній машині з різними портами:
- **Вузол 1 (основний)**: TCP 8000, UDP 9000 → запускається через DocControlService
- **Вузол 2 (тестовий)**: TCP 8001, UDP 9001 → запускається окремо

## 📋 Покрокова інструкція

### Крок 1: Запусти основний сервіс
1. Запусти **DocControlService** (Windows Service)
2. Запусти **DocControlUI**
3. Перейди на вкладку **"💻 Мережа"**

### Крок 2: Запусти тестовий вузол
Двічі клацни на файл:
```
start_test_node.bat
```

**Скрипт автоматично:**
- Перевірить наявність .NET SDK
- Зібере проект DocControlNetworkCore
- Створить папку C:\TestNode
- Скопіює exe та всі DLL залежності
- Створить конфігурацію з портами 8001/9001
- Запустить тестовий вузол

### Крок 3: Перевірка з'єднання
Через 10-20 секунд у UI має з'явитися:
- **Локальна система**: твій основний вузол
- **Доступні вузли**:
  - `TestNode@TEST-PC` (127.0.0.1:8001)

### Крок 4: Перегляд файлів
1. Клацни на тестовий вузол у списку
2. Права панель покаже файли з `C:\TestNode\SharedFiles`
3. Подвійне клацання на файл → завантаження

## 🔍 Що можна протестувати

### ✅ Виявлення вузлів (Discovery)
- Автоматичне виявлення через UDP broadcast
- Відображення у списку доступних вузлів
- Оновлення статусу (online/offline)

### ✅ Перегляд файлів
- Отримання списку файлів з віддаленого вузла
- Навігація по папках (подвійне клацання)
- Кнопки "⬆️ Вгору" та "🏠 На початок"

### ✅ Завантаження файлів
- Подвійне клацання на файл → діалог завантаження
- Кнопка "📥 Завантажити файл"
- Progress bar при завантаженні

### ✅ Інформація про вузол
- Кнопка "📋 Детальна інформація"
- Ping вузла для перевірки доступності

## 🛠️ Налагодження проблем

### Вузол не з'являється у списку
1. Перевір що обидва екземпляри запущені
2. Перевір Windows Firewall (UDP 9000, 9001)
3. Натисни "🔄 Оновити" у UI
4. Дивись логи в консолі обох вузлів

### Помилка при завантаженні файлів
1. Перевір що файли існують у `C:\TestNode\SharedFiles`
2. Перевір права доступу до папки
3. Дивись логи у консолі NetworkCore

### Вічне завантаження
1. Перезапусти обидва вузли
2. Перезапусти UI
3. Перевір що порти не зайняті іншими процесами:
   ```cmd
   netstat -ano | findstr "8000 8001 9000 9001"
   ```

## 📁 Структура папок після тестування

```
C:\TestNode\
├── DocControlNetworkCore.exe
├── network_identity.json       # Ідентифікатор тестового вузла
└── SharedFiles\                # Спільна папка
    ├── test1.txt
    └── test2.txt
```

## 🔄 Скидання до початкового стану

Видали папку:
```cmd
rmdir /s /q C:\TestNode
```

Або просто видали `network_identity.json` для генерації нового ID.

## ⚠️ Важливо

1. **Порти повинні бути різні** для кожного екземпляра
2. **SharedFiles папки різні** для кожного вузла
3. **InstanceId унікальні** (генеруються автоматично або вручну у JSON)
4. Обидва вузли мають бути **в одній мережі** (локальний loopback OK)

## 🎯 Очікуваний результат

При правильному налаштуванні ти побачиш:
- ✅ Два вузли виявляють один одного автоматично
- ✅ Можеш переглядати файли на обох вузлах
- ✅ Можеш завантажувати файли між вузлами
- ✅ Статус онлайн/офлайн оновлюється
- ✅ Навігація по папках працює

Це повністю імітує роботу в реальній мережі з двома комп'ютерами! 🎉
