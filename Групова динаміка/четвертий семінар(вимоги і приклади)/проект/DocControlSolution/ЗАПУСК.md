# 🐱 Coffee Cat - Document Control System v0.10.0

## 📋 Інструкція з запуску

### Варіант 1: Автоматичний запуск (РЕКОМЕНДОВАНО)

1. **Запустіть `StartDocControl.bat`** від імені адміністратора:
   - Правий клік → "Запустити від імені адміністратора"

2. Скрипт автоматично:
   - Перевірить стан Windows Service
   - Запустить сервіс (якщо потрібно)
   - Відкриє головне вікно програми

---

### Варіант 2: Ручний запуск

#### Крок 1: Запуск Windows Service

**Через Services.msc:**
1. Натисніть `Win + R` → введіть `services.msc`
2. Знайдіть `DocControlService`
3. Правий клік → "Запустити"

**Або через командний рядок (від адміністратора):**
```cmd
net start DocControlService
```

#### Крок 2: Запуск UI

Запустіть `DocControlUI.exe` з папки:
- `DocControlUI\bin\Debug\DocControlUI.exe` (для Debug)
- `DocControlUI\bin\Release\DocControlUI.exe` (для Release)

---

### Варіант 3: Тільки UI режим (без Windows Service)

Якщо не потрібні локальні функції, можна запустити тільки UI:

1. Запустіть `DocControlUI.exe`
2. У вікні запуску натисніть "Тільки UI режим"

**⚠️ В цьому режимі недоступні:**
- Локальні директорії
- Автоматичне версіонування (Git)
- AI аналіз

**✅ Доступні:**
- Мережевий браузер
- Підключення до інших пристроїв
- Віддалене редагування файлів
- Багатокористувацький режим

---

## 🔧 Компоненти системи

### 1. DocControlService (Windows Service)
- **Що робить:** Управляє локальними директоріями, версіонуванням, мережевими шарами
- **Потребує:** Права адміністратора
- **Обов'язковий:** Так (для повного функціоналу)

### 2. DocControlUI (WPF застосунок)
- **Що робить:** Графічний інтерфейс
- **Потребує:** Windows 10+ з .NET 8.0
- **Обов'язковий:** Так

### 3. Network Core (вбудований модуль)
- **Що робить:** TCP/UDP комунікація між пристроями
- **Потребує:** Відкриті порти 5000-5001
- **Обов'язковий:** Для мережевої роботи

---

## 🚀 Перший запуск

1. **Встановіть Windows Service** (один раз):
   ```cmd
   sc create DocControlService binPath= "C:\Path\To\DocControlService.exe"
   ```

2. **Запустіть `StartDocControl.bat`** від адміністратора

3. У вікні запуску натисніть:
   - "🚀 Запустити сервіс" (якщо не запущений)
   - "▶️ Продовжити" (якщо все готово)

4. **Додайте директорії:**
   - Вкладка "Директорії" → "Додати"
   - Вкажіть назву та шлях

5. **Надайте доступ іншим пристроям:**
   - Вкладка "Мережа" → оберіть пристрій
   - Оберіть директорію → "Надати доступ"

---

## 🌐 Багатокористувацький режим

### На Комп'ютері А (сервер):
1. Додайте директорію для роздачі
2. Надайте доступ пристрою "ComputerB"

### На Комп'ютері Б (клієнт):
1. Відкрийте вкладку "Мережа"
2. Знайдіть "ComputerA" у списку
3. Подвійний клік → відкриється браузер файлів
4. Відкрийте файл → редагуйте → зберігайте
5. **Автозбереження:** Кожні 10 секунд зміни відправляються на сервер

---

## ⚙️ Налаштування

### Порти (за замовчуванням):
- **TCP:** 5000
- **UDP:** 5001
- **Named Pipe:** DocControlPipe

### Інтервал автозбереження:
- Поточний: **10 секунд** (для тестування)
- Рекомендований: 30 секунд

### Heartbeat:
- **30 секунд** (підтримка блокування файлу)
- Автоматичне розблокування: через 5 хвилин неактивності

## 🐛 Вирішення проблем

### Проблема: "Windows Service не запущено"
**Рішення:**
1. Запустіть програму від адміністратора
2. Або запустіть сервіс вручну через `services.msc`

### Проблема: "Не підключається до сервісу"
**Рішення:**
1. Перевірте чи запущений `DocControlService`
2. Перевірте брандмауер (дозвольте Named Pipes)

### Проблема: "Не бачить інші пристрої"
**Рішення:**
1. Переконайтеся що пристрої в одній мережі
2. Перевірте брандмауер (дозвольте порти 5000-5001)
3. Вкладка "Мережа" → "Пошук вузлів"

### Проблема: "Зміни в Word не зберігаються"
**Рішення:**
- Це виправлено у v0.10.0.2
- Переконайтеся що використовуєте останню версію
